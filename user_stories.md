## Admin User Stories

As an admin, I want issues, so that I can:

1. Log into the portal with my username and password and manage the platform securely
2. Log out of the portal to protect system access
3. Add doctors to the portal
4. Delete doctor's profile from the portal
5. Run a stored procedure in MySQL CLI to get the number of appointments per month and track usage statistics

## Patient User Stories

As a patient, I want a portal interaction, so that I can:

1. View a list of doctors without logging to explore options before registering
2. Sign up using my email and password to book appointments
3. Log into the portal to manage my bookings
4. Log out of the portal to secure my account
5. Log in and book an hour-long appointment to consult with a doctor
6. View my incoming appointments so that I can prepare accordingly

## Doctor User Stories

As a doctor, I want to manage availability and appointments, so that I can:

1. Log into the portal to manage my appointments
2. Log out of the portal to protect my account data
3. View my appointments calendar to stay organized
4. Mark my unavailability to inform patients only the available slots
5. Update my profile with specialization and contact information so that patients have up-to-date information
6. View the patient details for upcoming appointments so that I can be prepared
